# Shortee OMX Brand Playbook

**Date:** 2026-03-08  
**Purpose:** Use `oh-my-codex` as a brand team operating system, not just a coding utility.

---

## 1. Brand Team Principle

브랜딩 팀은 "좋은 아이디어"를 만드는 팀이 아니라,  
**브랜드를 반복 가능하게 운영하는 팀**이다.

`oh-my-codex`는 이 반복 가능성을 만든다.

핵심 원칙:

- 한 에이전트가 다 잘하려 하지 않는다
- 역할을 분리한다
- 산출물은 문서와 자산으로 남긴다
- 매주 같은 리듬으로 돌린다

---

## 2. Recommended Brand Team Roles

### 1. Brand Strategist

실제 역할 매핑:

- `product-manager`

담당:

- 브랜드 포지셔닝
- 타깃 정의
- 브랜드 메시지 우선순위
- 캠페인 테마 결정

질문:

- 누구를 끌어와야 하는가
- 왜 우리를 기억해야 하는가
- 이번 달 무엇을 절대 하지 말아야 하는가

### 2. Brand Designer

실제 역할 매핑:

- `designer`

담당:

- 색 체계
- 랜딩/앱 화면 톤
- 카드 템플릿
- 광고 비주얼 룩

질문:

- 지금 화면이 generic한가
- 첫 3초 인상이 남는가
- 브랜드가 화면에서 말보다 먼저 보이는가

### 3. Brand Information Architect

실제 역할 매핑:

- `information-architect`

담당:

- 라벨 체계
- 탭 명칭
- 메뉴 구조
- 용어 통일

질문:

- 같은 개념이 다른 이름으로 불리고 있지 않은가
- 사용자가 지금 길을 잃는가
- 브랜드 언어가 구조에 반영됐는가

### 4. Brand Copy Lead

실제 역할 매핑:

- `writer`

담당:

- 앱 카피
- 광고 문안
- 앱스토어 문안
- 표현 카드 카피

질문:

- 이 문장이 Shortee처럼 들리는가
- 말이 너무 설명적이지 않은가
- 다른 영어 앱이 써도 이상하지 않은 문장인가

## 3. Operating Rhythm

### Monday

브랜드 전략 점검.

- 이번 주 핵심 메시지 1개
- 이번 주 밀 콘텐츠 축 2개
- 이번 주 금지할 카피/톤 3개

### Tuesday

비주얼 점검.

- 핵심 화면 톤
- 피드 카드 인상
- 공유 자산 템플릿

### Wednesday

카피 점검.

- 로그인
- 오늘 탭
- 페이월
- 공유 문구

### Thursday

캠페인 점검.

- 광고 후킹 문장
- SNS 카드
- 크리에이터 전달 문안

### Friday

브랜드 리포트.

- 이번 주 가장 브랜드다웠던 화면
- 가장 generic했던 표현
- 다음 주 교체할 요소

---

## 4. Weekly Deliverables

매주 최소 산출물:

1. 브랜드 메모 1개
2. 카피 교정안 1세트
3. 비주얼 가이드 수정안 1개
4. 캠페인 문안 10개
5. 금지어 목록 업데이트

---

## 5. Brand Review Framework

모든 결과물은 아래 5문항으로 검토한다.

1. **Shortee답나**
2. **장면 중심인가**
3. **공부 냄새가 덜한가**
4. **기억에 남는 한 줄이 있나**
5. **광고로 잘라 써도 되나**

3개 이하로 통과하면 버린다.  
4개면 개선 후 사용.  
5개면 바로 채택.

## 6. What Each Agent Should Produce

### product-manager

산출물:

- 브랜드 포지셔닝 메모
- 타깃별 메시지 맵
- 캠페인 우선순위 표

### designer

산출물:

- 색 체계 제안
- 핵심 화면 mood board
- 공유 카드 템플릿

### information-architect

산출물:

- 라벨 통일안
- 탭 구조안
- 용어 사전

### writer

산출물:

- 로그인/탐색/저장/공유 카피안
- 앱스토어 메타 문안
- 광고 카피 뱅크

## 7. Immediate Brand Tasks For OMX

`oh-my-codex`로 바로 굴릴 브랜딩 태스크:

### Task A: Naming cleanup

- `Shortee` 외부 명칭 고정
- `StudyEng` 잔존 텍스트 목록화
- 교체 우선순위 부여

### Task B: Tone cleanup

- 영어/한국어 혼용 라벨 제거
- 교육 앱처럼 들리는 문장 제거
- 브랜드 문장 통일

### Task C: Visual differentiation

- 보라 중심 기본 인상 탈피
- Shortee만의 포인트 컬러 확정
- 카드 / 버튼 / 탭의 무드 통일

### Task D: Shareable brand assets

- 표현 카드
- 장면 카드
- 친구 공유용 짧은 문장

## 8. Command Model For Brand Work

브랜딩 작업에서 우선적으로 써야 하는 사고 흐름:

1. `product-manager`로 왜 이 메시지가 필요한지 정리
2. `information-architect`로 어디에 어떤 단어를 쓸지 정리
3. `writer`로 실제 카피 제작
4. `designer`로 비주얼 인상 고정

즉, 브랜딩은:

**전략 -> 구조 -> 문장 -> 화면**

순으로 굴린다.

---

## 9. Brand Team Rules

### Rule 1

브랜드 팀은 "예쁘게" 만드는 팀이 아니다.  
**기억되게** 만드는 팀이다.

### Rule 2

기능 설명보다 브랜드 인상이 먼저 와야 한다.

### Rule 3

카피는 길어질수록 브랜드가 약해진다.

### Rule 4

한 번 정한 용어는 쉽게 바꾸지 않는다.

### Rule 5

광고용 문장과 앱 안 문장이 서로 다른 브랜드처럼 들리면 실패다.

---

## 10. Leadership View

브랜딩 팀장이 매주 대표에게 보고할 내용은 다섯 가지뿐이다.

1. 우리 브랜드를 한 문장으로 여전히 말할 수 있는가
2. 가장 강한 장면형 메시지는 무엇인가
3. 이번 주 가장 generic했던 화면은 무엇인가
4. 어떤 단어를 버렸는가
5. 다음 주 어떤 인상을 더 강하게 만들 것인가

이 다섯 가지가 분명하면 브랜딩 팀은 잘 굴러가는 것이다.
