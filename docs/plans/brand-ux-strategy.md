### 1-2. 경쟁사별 핵심 전략 분석

#### 케이크 (Cake) - 직접 경쟁사
- **핵심 강점:** "투데이" 탭으로 매일 접속 습관 유도. 무한스크롤로 탐색 부담 제거. 학습 단위가 "표현 1개"로 매우 가벼움.
- **UX 철학:** 검색 없이 훑어보기 → 매력적인 헤드카피로 클릭 유도 → 3회 반복 재생 → 퀴즈. 유저가 스스로 학습 취향을 발견하게 만듦.
- **차별화 포인트 vs StudyEng:** 케이크는 1개 표현 단위 학습이라 "깊이"가 부족. StudyEng은 숏폼 영상 전체를 경험시키는 것이 차별점이 될 수 있음.

#### 듀오링고 (Duolingo) - 간접 경쟁사
- **핵심 강점:** 게이미피케이션의 교과서. 스트릭 유지자는 Day-30 리텐션이 40% 높고, 전체 DAU 3,410만, MAU 1.3억에 달함. 80% 유저가 유기적으로 유입.
- **브랜드 전략:** 초록 부엉이 캐릭터를 기반으로 한 감정적 연결. TikTok/밈 마케팅으로 MZ세대 공감. 도발적이고 장난스러운 톤("연습 안 하면 부엉이가 슬퍼해요").
- **차별화 포인트 vs StudyEng:** 듀오링고는 텍스트 중심 + 구조화된 커리큘럼 = "공부 느낌"이 강함. StudyEng은 영상 기반 + 자유로운 탐색 = "놀이 느낌"에서 승부 가능.

#### 스픽 (Speak) - 방향성 참고
- **핵심 강점:** 세련된 디자인, AI 튜터 중심. "공부 안 하는 느낌"을 만드는 방법: 구조화된 레슨이지만 대화형으로 포장, 깔끔한 UI로 부담감 최소화.
- **브랜드 톤:** 프리미엄, 미니멀, 성인 대상. 이모지/캐릭터 거의 없음.
- **차별화 포인트 vs StudyEng:** 스픽은 스피킹 특화, StudyEng은 리스닝+표현 학습 특화. 스픽의 "깔끔하고 성숙한" 톤은 참고할 만함.

### 1-3. 콘텐츠 앱 홈 화면 트렌드 (Netflix, YouTube, TikTok)

2025-2026년 콘텐츠 앱의 핵심 트렌드는 "TikTokification"이다:

- **Netflix:** 2025년 대대적 UI 개편. 모바일에 TikTok 스타일 세로 피드 도입. "홈" (알고리즘 추천) + "My Netflix" (유저 큐레이션) 2개 섹션으로 재편. AI 검색 도입.
- **YouTube:** 이미 Shorts를 메인 탭에 통합. 홈은 개인화 추천 카드 그리드 + Shorts 진입점.
- **TikTok:** 앱 열면 즉시 풀스크린 콘텐츠 재생. 탐색 비용 0.

**핵심 인사이트:** 대형 콘텐츠 앱들이 모두 "세로 피드 + 즉시 콘텐츠 재생" 방향으로 수렴 중. StudyEng의 쇼츠 피드 접근은 트렌드에 부합하지만, 차별화를 위해 학습 맥락을 추가하는 것이 필요.

## 2. 브랜드 아이덴티티 제안

### 2-1. 현재 문제 진단

현재 StudyEng의 브랜드 아이덴티티는 **사실상 부재**한 상태:
- 색상: Tailwind 기본 `blue-500` (#3B82F6) + 순수 블랙/화이트 → 어떤 앱에서든 볼 수 있는 generic한 조합
- 이름: "StudyEng" → "Study" + "English"의 직역. 이름 자체가 "공부" 느낌을 줘서 핵심 철학("Play first")과 충돌
- 로고/타이포: 온보딩에서 "StudyEng" 텍스트를 그냥 표시할 뿐, 아이콘이나 심볼 없음
- 전체적으로 "개발자가 만든 프로토타입" 느낌

### 2-2. 브랜드 포지셔닝 매트릭스

```
                  재미/놀이
                    |
         듀오링고 --|-- StudyEng (목표)
                    |
     구조화 --------+-------- 자유로움
                    |
            스픽 ---|--- 케이크
                    |
                  진지/학습
```

**StudyEng의 포지션:** 재미 + 자유로움의 사분면. 케이크보다 더 놀이 같고, 듀오링고보다 더 자유롭고, 스픽보다 더 캐주얼한 위치.

### 2-3. 브랜드 톤 & 보이스

| 요소 | 방향 | 구체적 예시 |
|------|------|------------|
| **톤** | 쿨하고 담백한 (not 오글거리는, not 차가운) | "오늘도 한 편 봤네" (O) / "대단해요! 최고의 학습자!" (X) |
| **보이스** | 옆자리 친구 (not 선생님, not 코치) | "이 표현 쓸 일 많을 거야" (O) / "이 표현을 꼭 외워보세요!" (X) |
| **유머** | 건조한 위트 (not 이모지 폭탄, not 감성팔이) | "5초 남았는데 아직 안 골랐어" (O) / "화이팅! 할 수 있어요!" (X) |
| **격려** | 결과 중립적, 과정 인정 | "3개 맞았어. 나쁘지 않은데" (O) / "틀려도 괜찮아요, 다시 도전!" (X) |
| **압박** | 0 | 스트릭 끊겨도 "어제 안 왔네? 괜찮아" (O) / "스트릭이 사라졌어요!" (X) |

### 2-5. 앱 이름에 대한 의견

"StudyEng"이라는 이름은 "Play first, learn naturally" 철학과 직접적으로 충돌한다. "Study"라는 단어가 들어가면 사용자가 앱을 열 때 이미 "공부해야 한다"는 마인드셋이 됨.

장기적으로 리브랜딩을 고려할 필요가 있으나, 현재 MVP 단계에서는 우선순위가 아님. 이름보다는 앱 내 경험으로 "이건 공부가 아니라 놀이"라는 인식을 심는 것이 먼저.

## 3. 홈 화면 분석: 현재 vs 3가지 대안

### 3-1. 현재 구조 분석

현재 홈 화면은 앱을 열면 **즉시 TikTok 스타일 세로 쇼츠 피드**가 시작되는 구조.

| 장점 | 단점 |
|------|------|
| 진입 장벽 최소화 (탭 한 번 없이 콘텐츠 시작) | YouTube 영상 로딩 시간 (첫 화면 지연 1-3초) |
| TikTok/Shorts에 익숙한 유저에게 친숙 | "지금 뭘 보고 있는지" 맥락 부재 |
| 스와이프 = 다음 영상이라는 직관적 UX | 첫 영상이 마음에 안 들면 이탈 위험 |
| 몰입감 극대화 | 학습 상태(스트릭, 미션 등) 확인 불가 |
| 영상 위 + 자막 아래 분리 레이아웃으로 가독성 확보 | "이 앱이 뭘 하는 앱인지" 신규 유저에게 불명확 |

### 3-2. 대안 분석

#### 대안 1: Netflix 스타일 "카테고리 브라우징 → 선택 → 재생"

```
[오늘의 추천 히어로 배너]
[카테고리: 엔터테인먼트 | 뉴스 | 일상 | ...]
[추천 영상 가로 스크롤]
[시리즈 목록]
[전체 영상 그리드]
```

| 장점 | 단점 |
|------|------|
| 유저에게 선택권 부여 | 선택까지 탭 2-3회 필요 (진입 장벽 UP) |
| 학습 상태/대시보드 표시 가능 | 탐색 모드에서 "결정 피로" 발생 가능 |
| 콘텐츠 발견 경로가 다양 | 지금의 탐색(Explore) 탭과 역할 중복 |
| 신규 유저에게 앱 구조가 명확 | TikTok 세대에게 "올드한" 느낌 |

**판단:** 현재 **탐색 탭이 이미 이 역할**을 하고 있어, 홈을 이걸로 바꾸면 탐색 탭의 존재 의미가 사라짐. 부적합.

#### 대안 2: 케이크 스타일 하이브리드 홈

```
[인사 + 오늘의 미션 미니 카드]
[오늘의 추천 영상 카드 (1개, 탭하면 쇼츠 진입)]
[최근 이어보기 영상 카드]
[카테고리 바로가기]
```

| 장점 | 단점 |
|------|------|
| "투데이" 느낌으로 매일 접속 유도 | 콘텐츠까지 탭 1회 추가 |
| 학습 맥락 제공 (미션, 진행률) | 카드 기반 → 무한스크롤 유혹 → 케이크 카피 느낌 |
| 개인화된 첫 화면 가능 | 구현 복잡도 증가 |
| 신규/복귀 유저 모두에게 친절 | 쇼츠 피드의 "몰입감" 약화 |

**판단:** 좋은 접근이나, StudyEng의 핵심 차별점인 "즉시 영상 몰입"을 약화시킴. 부분 채택 가능.

#### 대안 3: 현재 유지 + 맥락 레이어 추가

```
[쇼츠 피드 (현재와 동일)]
+ 첫 진입 시 3초 환영 오버레이 → 자동 페이드
+ 하단 자막 영역에 미션 진행률 미니 표시
+ 상단에 학습 상태 미니 인디케이터 (스트릭 불꽃 등)
```

| 장점 | 단점 |
|------|------|
| 현재 강점(즉시 몰입) 유지 | 오버레이가 많아지면 산만 |
| 맥락/상태 정보 보완 가능 | 미션/상태가 "공부" 느낌 줄 수 있음 |
| 구현 비용 낮음 | 근본적 문제(로딩 시간) 해결 안 됨 |
| 기존 유저 경험 파괴 안 함 | |

**판단:** 가장 실용적인 접근. 단, 오버레이는 최소화해야 함.

### 3-3. 최종 추천: "Smart Feed" (대안 2 + 대안 3 혼합)

```
앱 열기
  |
  ├─ 첫 방문 or 복귀 유저 → 심플 대시보드 (3초 이내 스캔 가능)
  |   ├─ "Good morning / 오늘 2일째" (한 줄 인사)
  |   ├─ [오늘의 영상] 카드 (탭하면 쇼츠 피드 진입)
  |   ├─ [이어보기] 카드 (중단한 영상)
  |   └─ [오늘의 미션] 미니 프로그레스
  |
  └─ 쇼츠 피드 진입 후 → 현재와 동일한 풀스크린 영상 경험
      └─ 상단: 미니멀 상태 표시 (1/5 영상, 스트릭 아이콘)
```

**핵심 근거:**
1. **케이크의 "투데이" 전략** 차용: "오늘 새로운 콘텐츠가 있다"는 인식 = 매일 접속 동기
2. **즉시 몰입 유지:** 대시보드 → 영상 카드 탭 = 1탭으로 쇼츠 진입. 기존 대비 1탭만 추가.
3. **복귀 유저 맥락 제공:** "어제 뭘 봤지?"를 찾아 헤맬 필요 없음
4. **로딩 시간 은폐:** 대시보드를 보는 동안 첫 영상 프리로드 가능
5. **Netflix의 2025 인사이트 적용:** "홈 = 알고리즘 추천" + "마이 = 개인 큐레이션" 분리

**구현 우선순위:** Phase 2 (MVP 이후). 현재 MVP에서는 대안 3(오버레이 최소 보완)으로 충분.

## 4. 전체 UX 플로우 재설계 제안

### 4-1. 온보딩 재설계

**현재 문제:** 2스텝(브랜드 소개 → 레벨 선택)이 너무 간소. 유저의 관심사를 파악하지 못해 첫 영상 추천 정확도가 떨어짐.

**제안: 4스텝 (총 30초 이내)**

| 스텝 | 내용 | 목적 |
|------|------|------|
| 1. 브랜드 | 앱 로고 + "짧은 영상으로 영어가 재밌어져요" + [시작] | 앱 가치 전달 (3초) |
| 2. 레벨 | 초급/중급/고급 선택 | 콘텐츠 난이도 조절 |
| 3. 관심사 | "어떤 영상이 재밌어요?" + 카테고리 칩 3개 이상 선택 (엔터/뉴스/일상/음악 등) | 첫 피드 개인화 |
| 4. 즉시 체험 | 바로 첫 영상 재생 (자막 ON) + "스와이프해서 다음 영상" 툴팁 | 30초 내 핵심 경험 전달 |

**핵심 원칙:** 4스텝이지만 각 스텝이 5-8초. 마지막 스텝에서 바로 영상이 재생되어 "온보딩이 끝나고 앱이 시작되는" 경계를 없앰.

### 4-2. 탭 구조 재설계

**현재:** 홈(피드) / 탐색 / 학습 / 프로필 (4탭)

**제안:** 4탭 유지하되, 역할과 순서 재정의

| 탭 | 아이콘 | 역할 | 비고 |
|----|--------|------|------|
| **오늘** (기존: 홈) | Play 아이콘 | Smart Feed 대시보드 → 쇼츠 피드 진입 | 이름 변경: "홈" → "오늘" (케이크의 투데이 전략) |
| **탐색** | Search | 카테고리 브라우징, 시리즈, 검색, 전체 영상 | 현재와 동일 |
| **복습** (기존: 학습) | 반복 아이콘 | 저장한 표현, 게임, 시청 기록 | 이름 변경: "학습" → "복습" ("학습"은 공부 느낌) |
| **나** (기존: 프로필) | User | 스탯, 배지, 설정, 스트릭 | 이름 변경: "프로필" → "나" (더 캐주얼) |

**왜 4탭을 유지하는가:**
- 5탭 이상은 하단 네비게이션이 좁아져 터치 영역 축소
- 케이크는 5탭이지만, 스피크/클래스 분리가 StudyEng에는 불필요
- 현재 4탭의 역할 분담이 적절함. 이름만 바꾸면 인식이 달라짐

### 4-3. 탭별 상세 역할

#### "오늘" 탭
- **Phase 1 (현재):** 쇼츠 피드 직행
- **Phase 2:** Smart Feed 대시보드 → 쇼츠 진입
  - "오늘의 추천" 영상 카드 1개
  - "이어보기" (마지막 시청 중단 지점)
  - 미니 미션 프로그레스 (3개 중 1개 완료 같은)
  - 스트릭 카운터 (미니멀하게)

#### "탐색" 탭
- 현재 구조 유지 (카테고리 필터 + 시리즈 + 전체 영상)
- **개선점:**
  - 검색 기능 강화 (현재 SearchBar 있지만 활용도 낮음)
  - 시리즈를 더 눈에 띄게 (케이크의 "Path" 느낌으로 카드 확대)
  - "가장 많이 본 영상" / "새로 추가된 영상" 섹션 추가

#### "복습" 탭
- 현재: 데일리 미션 + 시청 기록 + 저장 표현 + 게임 → 잘 구성됨
- **개선점:**
  - 데일리 미션을 "오늘" 탭으로 이동 (접근성 향상)
  - 복습 탭은 저장 표현 + 게임에 집중
  - 표현 카드에 "마지막 복습: 3일 전" 표시 → 간격 반복 학습 유도
  - 게임 결과 히스토리 (어떤 표현을 틀렸는지)

#### "나" 탭
- 현재: 아바타 + XP + 스트릭 + 통계 + 배지 + 테마 + 설정
- **개선점:**
  - 주간/월간 리포트 (이번 주 시청 영상 수, 저장 표현 수, 게임 점수 추이)
  - "나의 영어 레벨" 변경 기능
  - 알림 설정 (시간대별 리마인더)
  - 앱 정보 / 피드백 보내기

### 4-4. 핵심 유저 저니

```
[첫 방문]
온보딩 (30초) → 첫 영상 자동재생 → 자막 탭해서 표현 저장
→ 2-3개 영상 시청 → 게스트 뷰 제한 → 로그인 유도

[일상 루프] (목표: 하루 5-10분)
앱 열기 → "오늘" 대시보드 스캔 (3초)
→ 오늘의 추천 or 이어보기 탭 → 쇼츠 피드 진입
→ 2-5개 영상 시청 → 표현 1-2개 저장
→ (선택) 복습 탭 → 게임 1판 (30초)
→ 미션 완료 확인 → 종료

[복귀 루프]
스트릭 리마인더 알림 → 앱 열기 → "어제 안 왔네? 괜찮아" 메시지
→ 일상 루프 반복
```

## 6. "오그라들지 않는 디자인" 원칙

### 6-1. 정의

"오그라드는 디자인"이란: 과하게 감성적이거나, 억지스러운 친절함이나, 유치한 표현으로 사용자가 민망함을 느끼는 디자인. MZ/Z세대에게 특히 거부감이 큼.

"오그라들지 않는 디자인"이란: 담백하고, 진정성 있고, 적당한 거리감을 유지하는 디자인. 쿨함과 따뜻함 사이의 균형.

### 6-2. 구체적 DO / DON'T

#### 카피라이팅

| DO | DON'T |
|----|-------|
| "오늘도 한 편 봤네" | "대단해요! 오늘도 열심히 공부하셨네요!" |
| "3개 맞았어. 나쁘지 않은데" | "아쉽지만 괜찮아요! 다시 도전해봐요!" |
| "어제 안 왔네? 괜찮아" | "스트릭이 끊어졌어요ㅠㅠ 다시 도전!" |
| "이 표현 쓸 일 많을 거야" | "이 표현을 꼭 외워보세요!! 화이팅!" |
| "새 영상 올라왔어" | "새로운 학습 콘텐츠가 업데이트되었습니다!" |
| "5초 남았는데 아직 안 골랐어" | "시간이 얼마 안 남았어요! 빨리 선택해주세요!" |
| "30일째" | "와! 30일 연속 학습 달성! 축하합니다!!" |

#### 시각 디자인

| DO | DON'T |
|----|-------|
| SVG 아이콘 (Heroicons 등) | 이모지 (특히 다수 나열) |
| 단색 배지 + 텍스트 | 화려한 그라데이션 배지 + 느낌표 |
| 미니멀 성공 애니메이션 (체크마크 + 페이드) | 폭죽/반짝이/별 폭발 |
| 숫자만 크게 (30일) | "30일 연속 학습 대단해요!" |
| 카드 모서리 radius: 16-20px | radius: 30px+ (너무 둥근 것 = 유치함) |
| 고정된 primary color 1개 | 무지개/그라데이션 남발 |

#### 인터랙션

| DO | DON'T |
|----|-------|
| 조용한 햅틱 피드백 | 화면 전체가 흔들리는 리액션 |
| 결과 즉시 표시, 빠른 다음 진행 | "결과를 분석 중이에요..." 기다림 |
| 토스트 2초 후 자동 사라짐 | 모달로 매번 "축하합니다!" |
| 스트릭 끊겨도 조용히 리셋 | "스트릭이 사라졌어요!" 모달 |
| 에러 시 구체적 안내 한 줄 | "오류가 발생했습니다. 다시 시도해주세요." 반복 |

#### 보상 시스템

| DO | DON'T |
|----|-------|
| XP +5 토스트 (미니멀, 2초) | "경험치 5점을 획득하셨습니다!" 모달 |
| 배지: 심볼 + 이름만 | 배지: 별+왕관+불꽃+이모지 조합 |
| 레벨업: 숫자 변화 + subtle 애니메이션 | 레벨업: 전체 화면 축하 파티 |
| "새 배지: 연속 7일" (사실만 전달) | "와! 7일 연속 학습 배지를 획득!" |

### 6-3. "쿨함"의 기준

1. **절제:** 하고 싶은 말의 50%만 한다
2. **자신감:** 느낌표 대신 마침표. "잘했어요!"가 아니라 "잘했네."
3. **진정성:** 거짓 칭찬보다 건조한 팩트. "3/5 맞았어"
4. **여백:** 화면의 30%는 비운다. 빼곡한 화면 = 압박감 = 공부 느낌
5. **일관성:** 톤이 갑자기 바뀌지 않는다. 조용했다가 갑자기 "축하!!" 하면 오그라듦

| 4 | **빈 상태 카피 점검** | "영상을 보고 표현을 모아보세요" | "오그라들지 않는" 톤으로 전체 교정 | 곳곳에 "해보세요!", "할 수 있어요!" 패턴 |
| 5 | **로딩 상태 개선** | 검은 화면 + 스피너 | Skeleton UI 또는 브랜드 로고 펄스 | 첫 로딩 3초가 이탈 결정 |

### 무료→유료 전환을 위한 UX 패턴 (우선순위순)

1. **Soft Paywall with Metered Stop:** 현재 하루 5편 제한은 좋은 전략이나, "5편 다 봤어요" 메시지가 아니라 "오늘 5편 봤네. 내일 또 만나" 톤으로. 프리미엄 제안은 부드럽게.

2. **Preview, Then Pay:** 한국어 자막을 1-2개 영상에서 무료 체험 → 이후 프리미엄. "이게 있으면 이해가 확 되는구나" 경험 후 제한.

3. **Earned Discount:** 현재 미션 달성률 기반 할인은 훌륭한 전략. 이를 더 전면에 노출. "이번 달 미션 80% 달성하면 30% 할인" → 활동량 + 전환률 동시 상승.

4. **Trial Trigger:** 앱 사용 7일째에 "3일 프리미엄 체험" 자동 제공. 체험 후 유료 전환이 3x 높음.

5. **Annual Plan Anchoring:** 현재 연간 79,900원 vs 월간 9,900원. 연간 대비 월간의 "비쌈"을 시각적으로 강조. "월 6,658원" 등 환산 가격 표시.
